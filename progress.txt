## Codebase Patterns
- **CRITICAL: Database uses Supabase with postgres-js driver. Do NOT change src/db/index.ts to use @neondatabase/serverless - it will break the connection.**
- Use `npx shadcn@latest add <component>` to add new UI components
- Format with `npm run format` before committing
- Run `npm run typecheck` and `npm run lint` for quality checks
- Project uses single quotes, semi-colons, and 100 char line width (Prettier config)
- Database schema lives in `src/db/schema.ts`, export types using `$inferSelect` and `$inferInsert`
- Use `DATABASE_URL=... npm run db:generate` to test migrations without real DB
- Clerk middleware in `src/middleware.ts` protects /dashboard/* routes; add new protected routes to `isProtectedRoute` matcher
- Auth pages use catch-all routes: `/sign-in/[[...sign-in]]/page.tsx` and `/sign-up/[[...sign-up]]/page.tsx`
- Webhook endpoints at `/api/webhooks/*` are public (not protected by auth middleware)
- Use route groups `(public)` and `(dashboard)` for separate layouts; route groups don't affect URL paths
- Navigation components live in `src/components/navigation/` and are exported via index.ts
- Use `usePathname()` from next/navigation for active link highlighting in client components
- Sheet component from shadcn is good for mobile slide-out menus
- Zod validation schemas go in `src/lib/validators/` and export types using `z.infer<typeof schema>`
- Server actions use 'use server' directive at top of file; can return discriminated unions for success/error
- React Hook Form with shadcn uses Form/FormField/FormItem/FormControl/FormMessage pattern
- Onboarding routes at /onboarding/* are protected by auth middleware (same as /dashboard/*)
- Dynamic route params in Next.js 14+ are Promise: use `const { slug } = await params` in page/generateMetadata
- Public pages go in `(public)` route group; coach profile display components live in `src/components/coaches/`
