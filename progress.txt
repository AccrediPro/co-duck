## Codebase Patterns
- Use `npx shadcn@latest add <component>` to add new UI components
- Format with `npm run format` before committing
- Run `npm run typecheck` and `npm run lint` for quality checks
- Project uses single quotes, semi-colons, and 100 char line width (Prettier config)
- Database schema lives in `src/db/schema.ts`, export types using `$inferSelect` and `$inferInsert`
- Use `DATABASE_URL=... npm run db:generate` to test migrations without real DB
- Clerk middleware in `src/middleware.ts` protects /dashboard/* routes; add new protected routes to `isProtectedRoute` matcher
- Auth pages use catch-all routes: `/sign-in/[[...sign-in]]/page.tsx` and `/sign-up/[[...sign-up]]/page.tsx`
- Webhook endpoints at `/api/webhooks/*` are public (not protected by auth middleware)
