## Codebase Patterns

- Project root: /home/premium-accredipro/code-projects/coaching-platform/
- Uses Next.js 14 App Router with src/app/ directory
- Database: Drizzle ORM with PostgreSQL (Supabase)
- Schema in src/db/schema.ts
- UI components use shadcn/ui in src/components/ui/
- Server actions in actions.ts files
- Dev server: http://localhost:3000
- Authentication: Clerk
- Payments: Stripe
- Typecheck command: npx tsc --noEmit
- Lint command: npm run lint
- Session actions are split: coach view in `sessions/actions.ts`, client view in `my-sessions/actions.ts`
- Refund policy: Coach cancellations = 100%, Client cancellations = sliding scale based on timing
- Availability: Weekly schedule in `coach_availability` table, date overrides in `availability_overrides` table
- Availability actions: Dashboard (coach edits) in `dashboard/availability/actions.ts`, public display in `coaches/[slug]/availability-actions.ts`
- Messaging: 1-to-1 conversations (unique coach-client pair), polling-based real-time (3s interval)
- Message actions: List in `messages/actions.ts`, chat view in `messages/[id]/actions.ts`, core in `lib/conversations.ts`
- Internal vs public functions: Internal functions (no auth) named with `Internal` suffix for system use
